# OKR撰写助手

## 适用平台
ChatGPT / Claude / Kimi / DeepSeek / 通义千问 / 豆包

## 使用方法
复制下面的提示词，粘贴到 AI 对话框，把「」里的内容换成你自己的。

## 提示词

```
你是一位 OKR（目标与关键结果）专家。请根据我的岗位和工作重点，帮我制定 OKR。

我的岗位：「如：产品经理」
考核周期：「如：2026年Q3」
团队/公司目标：「如：提升用户留存率到60%」
我的核心职责：「如：负责用户增长和体验优化」

请按以下格式输出：

## Objective 1：「用一句话描述目标」
**Key Results：**
- KR1：「可量化的关键结果」
- KR2：「可量化的关键结果」
- KR3：「可量化的关键结果」

## Objective 2：...
（2-3 个 Objective 即可）

要求：
- 每个 Objective 对应 3 个 Key Result
- KR 必须可量化（百分比、数字、时间节点）
- O 要有挑战性但可达（不要"维持日常运营"）
- KR 要有 70% 的信心能完成
- 每个 KR 说明为什么选它（一句话）

不要写"提高工作效率""加强团队协作"这种无法衡量的 O。
```

## 示例

**输入：** 后端开发工程师，Q3，公司目标是系统稳定性 99.9%

**输出：**
```
## Objective 1：核心服务可用性从 99.5% 提升到 99.9%
**Key Results：**
- KR1：所有核心接口 P99 延迟降低 50%（目前 800ms → 400ms 以下）
  原因：上半年用户投诉最多的就是慢
- KR2：线上故障 MTTR（平均恢复时间）从 45 分钟降到 15 分钟以内
  原因：大部分时间浪费在"谁在排查"，不是"修不好"
- KR3：实现核心链路 100% 自动告警 + 降级预案
  原因：目前 6 个核心接口有 2 个没有告警

## Objective 2：技术债清理，代码质量可视化
...
```

## 技巧
- OKR 不是 to-do list，别把日常工作写成 O
- 如果你写出来的 KR 自己都觉得"肯定完不成"，拆成两步
- 季度中间复盘一次，歪了赶紧调方向
